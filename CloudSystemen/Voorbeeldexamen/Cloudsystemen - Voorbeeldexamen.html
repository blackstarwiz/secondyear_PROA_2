Docker images (5 punten)

Je krijgt een eenvoudige applicatie geschreven in C♯, met het framework ASP.NET Core.

Verpak de applicatie in een Docker image (door er een Dockerfile voor te schrijven). Gebruik hiervoor de image dotnet-sdk van Docker Hub user bitnami. Je gebruikt versie 8-debian-12.

Het commando om een .NET package te installeren is dotnet add package packagenaam. De package voor ASP.NET Core heet Microsoft.AspNetCore.App. ASP.NET Core gebruikt standaard poort 8080. Het commando om een ASP.NET Core applicatie uit te voeren is dotnet run --urls http://0.0.0.0:8080.

Test dat je de applicatie lokaal kan bezoeken. Als dat niet meteen lukt, denk er dan aan dat je kan inloggen op een lopende container en daar commando's kan uittesten voor je ze in je Dockerfile plaatst.

Tag je eigen image als volgt: docker tag <lokaleimagenaam> <jouwdockerhubusername>/<imagenaam>. Zet je image op Docker Hub. Hou alle nodige stappen bij in een tekstbestand. Schrijf zinnen met daarin zowel de commando's als de reden waarom je ze gebruikt en waarom je ze zo schrijft. Alle bronfiles en je Dockerfile horen onder de map vraag1 van het startbestand.

Docker Compose (5 punten)

Je krijgt een simpele VueJS-applicatie. Het maakt niet uit of je VueJS kent. Deze applicatie is gebouwd bovenop NodeJS, je kan ze opstarten met npm run serve en ze luistert standaard op poort 5173. Je hoeft niets aan de werking van de applicatie aan te passen. Je kan zorgen dat deze applicatie bereikbaar is door de Docker Compose file aan te vullen. Maak ze bereikbaar op poort 8080 (zorg dan wel dat de container uit vraag 1 niet aan staat).

Deze applicatie veronderstelt dat er een API aangeboden wordt voor “note”-items (dus notities die bijgehouden worden in JSON-formaat). Hiervoor gebruik je de image vimagick/json-server. Containers op basis van deze image bieden een API aan op containerpoort 3000. Daarvoor verwachten ze wel dat er een JSON file genaamd database.json bestaat in de map /data.

Wanneer je de VueJS applicatie bezoekt, wordt de API vanzelf gecontacteerd op http://localhost:3000/notes. Dit zal niet vanzelf werken, maar kan ook opgelost worden via de Compose file.

Tip: gebruik de developer tools van je browser om beter te begrijpen wat er gebeurt.

Github Actions (4 punten)

Je krijgt enkele startbestanden. Je moet hier een publieke GitHub repository voor maken. Gebruik dan de action codecov-action van de gebruiker codecov om te zorgen dat, bij een wijziging op de main branch, een code coverage rapport wordt aangemaakt geüpload. Je zou moeten weten waar je de documentatie hierrond terugvindt zonder een zoekmachine te gebruiken.

Codecov vereist dat je een account aanmaakt. Dit is gratis. Doe dit door de account te koppelen aan je Github account. Tijdens dit proces zal gevraagd worden een “Github app” te installeren. Je moet dat inderdaad doen. Het gaat niet om software die je op jouw machine moet installeren, maar om software die toegang krijgt tot je repository.

Op de site van Codecov (die je mag raadplegen) moet je de naam van je Github repo ingeven in deze search box:

Coverage uploaden vereist een access token en dat vind je onder “Settings” → “Repository upload token”.

Traefik (6 punten)

Maak een nieuwe Docker Compose file waarin je de .NET applicatie van vraag 1 beschikbaar maakt via Traefik. Je maakt gebruik van de image die je naar docker hub geupload hebt.

Zorg dat de .NET applicatie bereikbaar is op http://website.localhost/helloworld. Gebruik Traefik om deze routering in te stellen.

Voeg basic authentication toe. Enkel wie een geldige gebruikersnaam en wachtwoord ingeeft, mag toegang krijgen tot de applicatie.

Je zal een statische configuratie nodig hebben (in een traefik.yml bestand of via command-line args) en een dynamische configuratie (via labels in je Compose file of een aparte file).

Voor basic authentication heb je een gebruikersnaam en een geëncrypteerd wachtwoord nodig. Die kan je genereren met de htpasswd tool.

Alle nodige configuratiebestanden horen onder de map vraag4 van het startbestand.
